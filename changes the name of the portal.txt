doctor portal update 
1️⃣ When doctor searches:

A GET request is sent to /getPatientHistory?email=patient@gmail.com

Server reads records.json → filters all records for that email → sends back as a list.

2️⃣ When doctor adds a record:

A POST request goes to /addRecord

Server stores new data (like diagnosis, doctor name, date) inside records.json
